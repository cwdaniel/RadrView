# Configuration

## Global Environment Variables

All services read from the same set of environment variables (set in `docker-compose.yml` or `.env`).

| Variable | Default | Description |
|---|---|---|
| `REDIS_URL` | `redis://localhost:6379` | Redis connection URL. In Docker Compose, use `redis://redis:6379`. |
| `DATA_DIR` | `./data` | Root directory for MBTiles tile storage. In Docker, bind-mounted as `/data`. |
| `PORT` | `8600` | HTTP port the tile server listens on. |
| `ZOOM_MIN` | `2` | Minimum tile zoom level generated by the tiler and compositor. |
| `ZOOM_MAX` | `10` | Maximum tile zoom level generated by the tiler and compositor. Radar data resolution does not justify beyond 10 for most sources. |
| `RETENTION_HOURS` | `24` | How many hours of frame data to retain. The cleanup worker deletes frames older than this. |
| `LOG_LEVEL` | `info` | Pino log level. One of: `trace`, `debug`, `info`, `warn`, `error`, `fatal`, `silent`. |
| `POLL_INTERVAL_MS` | `30000` | How often MRMS ingesters poll for new data (milliseconds). Default is 30 seconds. |

## NEXRAD-Specific Variables

| Variable | Default | Description |
|---|---|---|
| `NEXRAD_ENABLED` | `true` | Set to `false` to disable all NEXRAD Level 2 ingestion. MRMS composite tiles will be used at all zoom levels. |
| `NEXRAD_ZOOM_MIN` | `8` | Minimum zoom level at which NEXRAD Level 2 tiles are generated and composited. Below this zoom, MRMS tiles are always used. |
| `NEXRAD_STATIONS` | `all` | Comma-separated list of WSR-88D ICAO station codes to ingest (e.g. `KLOT,KIWX,KGRR`). Use `all` to ingest all 159 stations. |

**Notes:**
- `NEXRAD_ZOOM_MIN=8` is recommended. Setting it lower than 8 is wasteful; MRMS 1 km data is sufficient at z2–z7.
- Ingesting all 159 stations at full resolution requires more CPU and storage than the MRMS path. Start with a regional subset using `NEXRAD_STATIONS` if resources are constrained.
- Real-time sweep display is automatic for any station that publishes to `unidata-nexrad-level2-chunks`. No additional configuration is required.

## MRMS-Specific Variables

The MRMS ingester (`dist/ingest/mrms.js`) reads two additional variables:

| Variable | Default | Values | Description |
|---|---|---|---|
| `MRMS_REGION` | `conus` | `conus`, `alaska`, `hawaii` | Which MRMS regional S3 prefix to poll. |
| `MRMS_PRODUCT` | (region default) | `type` | Set to `type` to ingest `PrecipFlag_00.00` instead of the default reflectivity product. Outputs to the `{source}-type` source name. |

**Region defaults:**

| Region | Default Product | S3 Prefix | Bounds |
|---|---|---|---|
| `conus` | `SeamlessHSR_00.00` | `CONUS` | 20°N-55°N, 130°W-60°W |
| `alaska` | `SeamlessHSR_00.00` | `ALASKA` | 50°N-75°N, 180°W-120°W |
| `hawaii` | `MergedBaseReflectivity_00.50` | `HAWAII` | 15°N-25°N, 165°W-150°W |

## Docker Compose Per-Service Variables

Each service in `docker/docker-compose.yml` sets its own environment. Key differences:

### Ingest services (`ingest-mrms`, `ingest-mrms-alaska`, etc.)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - LOG_LEVEL=info
  - POLL_INTERVAL_MS=30000
  - MRMS_REGION=conus          # alaska or hawaii for regional services
  - MRMS_PRODUCT=type          # only on *-type services
```

### Tiler services (`tiler-1`, `tiler-2`)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - ZOOM_MIN=2
  - ZOOM_MAX=10
  - LOG_LEVEL=info
```

### Compositor (`compositor`)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - ZOOM_MIN=2
  - ZOOM_MAX=10
  - LOG_LEVEL=info
```

### Server (`server`)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - PORT=8600
  - LOG_LEVEL=info
```

### Cleanup (`cleanup`)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - RETENTION_HOURS=24
  - LOG_LEVEL=info
```

### NEXRAD ingester (`ingest-nexrad`)

```yaml
environment:
  - REDIS_URL=redis://redis:6379
  - DATA_DIR=/data
  - LOG_LEVEL=info
  - NEXRAD_ENABLED=true
  - NEXRAD_ZOOM_MIN=8
  - NEXRAD_STATIONS=all       # or e.g. KLOT,KIWX,KGRR
```

## Enabling and Disabling Sources

Sources are enabled by running their Docker Compose service. To disable a source, remove or comment out its service block in `docker/docker-compose.yml`.

**Minimum viable setup (MRMS CONUS only):**

```yaml
services:
  redis:        # required
  ingest-mrms:  # at least one ingester
  tiler-1:      # at least one tiler
  compositor:   # required for composite source
  server:       # required
  cleanup:      # recommended
```

**Disable DWD** (EU radar): remove or comment out `ingest-dwd` service. The compositor will simply have no DWD data and `composite-eu` will be empty until you re-enable it.

**Disable type sources**: remove all `*-type` services. The `precip-type` palette will fall back to the `default` palette when no type tile is available.

## Zoom Level Notes

- `ZOOM_MIN=2` — world overview. Very few tiles at this level; cheap to generate.
- `ZOOM_MAX=10` — approximately 152m/pixel at the equator. This matches MRMS 1km resolution well (oversampled but not wasteful). MRMS tiles are generated up to this level.
- NEXRAD Level 2 tiles are generated separately at z8–z14 (controlled by `NEXRAD_ZOOM_MIN`). They are stored under `nexrad/{stationId}` and composited at the appropriate zoom level, independent of `ZOOM_MAX`.
- There is no longer a GPU upscaler in the stack. Native 250m NEXRAD data provides full-resolution tiles at z8+ without upscaling.
- All services that generate MRMS tiles (`tiler-*`, `compositor`) must agree on `ZOOM_MIN`/`ZOOM_MAX`. The NEXRAD ingester uses its own zoom range controlled by `NEXRAD_ZOOM_MIN`.
